前端基础

发表于 2022-09-21 更新于 2022-09-28 分类于 前端 Waline： 阅读次数： 本文字数： 4.9k 阅读时长 ≈ 4 分钟

前端关注点：美观、功能、无障碍、性能、兼容、安全、用户体验

前端三件套，html+css+JavaScript

html

html，hypertext markup language，超文本标记语言

DOM树、DOM节点

  1. 一个完整的URL由 schme+host+path+query+hash组成，hash就是在页面不变的情况下，对于浏览数据的位置变动
  2. 跳转属性：target=’_self’，当前窗口打开； target=’_blank’，在新窗口打开
  3. pre标签，保证标签内空格不被合并
  4. 实体字符：entity，，&nbsp 空格 &lt 小于号等
  5. 视音频标签：video标签是html自带的视频标签，有src、controls、autoplay、muted等属性，track标签是视频字幕标签；audio是自带的音频标签
  6. 页面标签：一个页面可以分header、main、aside和footer四个页面标签，article 和section也是两个对应的内容标签
  7. 列表标签：<ul> unorder-list ，<ol> order-list，<li> list-item
  8. 导航标签：<nav>
  9. 表格标签：<thead>表头，<tbody>表格内容，<tr> 表行，<th> 表头单元格，<td> 表格数据单元格；rowspan属性，跨越多行，colspan属性，跨越多列
  10. 表单标签：form；input的type有text，password，number，date，file等；textarea，多行文本框，rows cols属性指定文本框最大行列数
  11. 表单单选框，input的type=radio，name属性决定了哪些互斥；checkbox多选框，checked属性是否默认选中
  12. 下拉选择框，select，下来选择项，option
  13. link 加载预先资源
  14. microdata

根据内容选择合适的标签！

css简介

css，cascading style sheets，用来定义页面元素的样式，页面中使用css主要分为外联<link>、嵌入<style>和内联<p style=’’> </p>三种

选择器基础

  1. * 是通配选择器； #id，id选择器；.class，class类名选择器；

  2. :root 根选择器

  3. 标签[属性]，属性选择器，带有该属性的都被选择中，eg a[title] {}

    1. a [ href= ‘https://www.baidu.com'] {}
    2. ^= 搜寻以什么开头的值 a [ href ^= ‘https’] {}
    3. $= 搜寻以什么结尾的值 a [ href ^= ‘.com’] {}
    4. *= 搜寻包含关键字的值 a [ href *= ‘baidu’] {}

  4. 伪类选择器，对于普通选择器加以修饰，可分为动态伪类和结构性伪类，

    • a:visited{} 浏览过的
    • a:hover {} 悬浮在上面的
    • a:nth-child(even) {} 选取偶数
    • 更多用户请看官方文档

  5. 选择器可彼此组合，

    1. 直接组合 AB，满足A且满足B；
    2. 后代组合A B，满足A的所有B；
    3. 亲子组合，A>B，只选择A子元素的B
      • A B 和 A>B 的区别在于 **子子元素 **是否被选择
    4. .container + div 意思为 选取拥有container这个class的html元素与它身处同一层紧接着的div
    5. .container ~ div 意思为 选取拥有container这个class的html元素与它身处同一层之后所有的div，即使中间有其他元素隔开

  6. 选择器组 h1, h2, h3 { … }

  7. css优先级，#id 大于 .class 大于 元素或伪元素

  8. css会自动继承父元素计算值，除非另外设置

  9. em是相对长度单位，并不固定，继承父级字体大小；px是像素数

盒模型基础

  1. margin 外边距在垂直方向上会采取margin collapse策略，即不合并，而取最大值；
  2. box-sizeing设置为border-box时，长宽则会是包含margin和padding的大小
  3. 当content内容溢出，可以通过设置overflow属性设置超出部分是 visible可见\ hidden不可见\ scroll滑动可见
  4. 盒子有块级盒子和行级盒子之分，块级盒子block 不和其他盒子并列摆放，行级盒子inline和其他行盒放在一行或拆开成多行，盒模型中的width、height不适用于行盒
  5. 块级元素生成块盒，body、article、div、main、section、h1-6…
  6. 行级元素生成行盒，span、em、strong、cite、code….
  7. display属性分为block、inline和inline-block，其中inline-block被放在行盒，但可以设置宽度，且不被拆分
  8. 盒子效果，border-radius盒子边角度
  9. 背景生效位置，background-clip，border-box全盒生效；padding-box，内边距和内容生效；content-box，仅在内容生效
  10. 行内元素垂直对齐方式，vertical-align

css的单位

  1. px是绝对单位
  2. em是相对单元，基于目前容器的大小进行设定，可以理解为倍数
  3. rem和em类似，但是它是基于root的html元素的大小设定去计算
  4. vw & vh，全程是 viewport width和viewport height，即用户看到的窗口，值在0 ~100之间
  5. vmin & vmax，vmin代表屏幕较短的一边，vmax代表屏幕较长的一边，概念与vw vh类似，常用于手机旋转屏设备

css布局方式

  • 常规流：行级、块级、flexbox、grid布局
  • 浮动
  • 绝对定位

常规流 normal flow

除了根元素、浮动和绝对定位元素不在，其他都在常规流之内

行级排版上下文，Inline Formatting Context IFC，水平排版

块级排版上下文，Block Formatting Context BFC

flexible box

  1. display: flex 生成块级flex容器；display: inline-flex 生成行级flex容器；

  2. flex-diretion控制流向，默认是从左到右，flex-diretion的方向称为主轴，与其垂直的方向称为侧轴，默认是row

    • row，column，row-reverse，column-reverse

  3. justify-content 控制元素主轴方向排版

    • flex-start、flex-end、center、space-between、space-around、space-evenly

  4. align-items 控制元素侧轴方向排版

    • flex-start、flex-end、center、stretch、baseline

  5. align-content 控制 **多行内容 **在侧轴方向排版

    • flex-start、flex-end、center、stretch（默认值，拉伸到父容器）、baseline（元素统一基准线）

  6. flex-wrap 控制是否允许换行，默认是nowrap不换行

  7. flex-flow 是flex-direction和flex-wrap的缩写，flex-flow：row nowrap

  8. flex容器有收缩的能力

    • flex = flex-grow + flex-shrink + flex-basis
    • flex-grow 控制有剩余空间时flex-item的放大比例；0表示默认大小
    • flex-shrink 控制空间不足时flex-item的缩小比例；
    • flex-basis 主轴方向的基础长度，auto刚好包裹元素

  9. order 控制 DOM元素出现顺序，从小到大，默认都是0

  10. align-self 控制覆写单个元素 的侧轴排版

    • flex-start、flex-end、center、stretch、baseline

grid box

flex将元素按照单向进行摆放，grid则可以将元素按照两个方向进行摆放

  1. display：grid生成块级容器；grid-template将容器划分为网格，再设置每一个子项占据哪些行、列
  2. grid-template-columns 控制网格列划分，grid-template-rows 控制网格行划分
    • grid-template-columns: 100px 100px 100px，也可以写为repeat(3,100px)
    • grid-template-columns: 30% 30% auto
    • grid-template-columns: 100px 1fr 1fr 说明：fr，fraction 份数，即剩下的均分比例
  3. grid-area 控制元素在网格中的位置，grid-area：1/1/3/3 左上角1,1和右下角3,3
  4. grid-areas 控制各个网格的名字，grid-areas: ‘header header header header’，repeat不适用于此
  5. grid-gap 控制网格间距
  6. justify-items 控制元素在网格内水平方向上的布局；align-items 控制垂直方向上的布局
  7. align-self，justify-self 控制单个元素的布局
  8. justify-content 控制水平方向 网格总体布局，align-content 控制垂直方向 网格总体布局

浮动 float

  1. 浮动元素脱离常规流，漂浮在容器左边或右边；
  2. 浮动元素贴着容器边缘或其他浮动元素
  3. 浮动元素不会影响常规流中的块级盒子
  4. 浮动元素后面的行盒会变短以避开浮动元素
  5. clear 属性控制元素受不受浮动元素影响

绝对定位 position

  1. position有四个取值：
    • static、默认值，非定位元素；
    • relative、相对自身原本位置偏移，不脱离文档流；
    • absolute，相对于最近的非static祖先进行绝对定位；
    • fixed，相对于浏览器窗口的绝对定位
  2. position：relative，使用top、left、bottom、right设置偏移长度，流内其他元素当它没有偏移一样进行布局，即使会有遮挡
  3. position：absolute，脱离常规流，不影响流内元素布局
  4. position：fixed，相对于窗口定位，不随页面滚动而变化，常用于导航栏

堆叠层级

z-index，数值越大，越靠近用户

css transform 变形

对元素进行平移 translate、旋转 rotate、缩放 scale、倾斜 skew，transform不会对其他元素布局产生影响

  • transform ：translate(100px, 100px)
  • transform ：rotate(90deg) 顺时针旋转90度

css transition 过渡

transition-property 需要过渡效果的属性；transition-duration 过渡耗时；transition-timing-funtion 过渡效果；transition-delay 过渡延迟

  • transition：height 500ms linear 1s
  • transition-timing-funtion 有以下几种：linear、ease、ease-in、ease-out、ease-in-out、steps(4)

css animation 动画

响应式设计，同一个页面适应不同屏幕大小设备的方案

  1. 设置viewport，<meta name=’viewport’ content=’width=device-width’, initial-scale=1.0’>
  2. contain 和 cover 的区别：contain，不造成裁剪，可能有留白；cover，可能会造成裁剪，不留白
  3. media query 针对不同屏幕，应用不同样式，@media screen and (min-width:480px){ } 在至少480px的屏幕上才使用该样式
    • 可以查询的media，width/height，device-width/device-height，device-pixel-ratio、orientation

浏览器hack层叠原理

  1. 同一个属性，后书写的值会覆盖前面书写的值
